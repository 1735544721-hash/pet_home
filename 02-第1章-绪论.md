第 1 章 绪论

1.1 研究背景

近年来宠物经济持续升温，领养、寄养、训练与宠物用品消费等服务形态不断丰富，线上预约、服务评价与健康档案等业务逐步从线下纸质记录转向数字化管理，对信息一致性、流程协同与服务体验提出更高要求[1]。宠物服务机构在日常运营中涉及宠物信息、用户身份、服务项目、订单支付与售后评价等多类数据，业务链条长且状态变化频繁，缺乏统一管理平台容易导致信息割裂与数据口径不一致，影响服务闭环的形成[2]。

一方面，领养与寄养流程常依赖多渠道沟通与手工登记，申请审核、预约排期与服务交付之间缺少同步机制，造成记录重复、状态滞后与责任界定模糊；另一方面，训练课程、服务预约与商品订单通常由不同模块或人员维护，难以统一追踪订单与服务履约，统计分析与客户回访成本较高；同时，宠物健康记录与疫苗信息分散存储，缺乏持续档案会削弱服务机构对安全与风险的管控能力。

1.2 研究意义

从实践应用角度看，建设面向宠物之家业务的一体化管理系统能够将宠物信息、领养寄养、训练预约、服务订单与健康记录等数据统一纳管，支持用户在线申请、预约排期、订单支付与评价反馈在同一平台完成，减少人工沟通与重复登记，提升服务流程的连续性与可追溯性[3]。

从管理价值角度看，系统通过用户与权限管理、服务状态维护与日志留痕，帮助机构实时掌握预约负载、订单履约与评价情况，便于运营人员进行资源调度、服务质量监控与异常预警，同时为公告发布、营销活动与统计报表提供稳定的数据基础[4]。

从理论研究角度看，本研究围绕宠物服务业务的核心实体与流程关系构建数据模型与功能链路，形成针对领养寄养、服务预约与订单闭环管理的需求分析与模块划分方法，并以 Spring Boot 3.2、MyBatis-Plus 与 Vue 3 构建前后端分离架构，为同类宠物服务管理系统的工程化实现提供可复用的设计参考。

1.3 国内外研究现状

1.3.1 国内研究现状

国内关于宠物服务管理的信息化研究多以宠物医院、宠物门店或综合服务平台为对象，关注宠物档案、疫苗记录、服务预约与收费结算等基础数据的结构化管理，并采用 B/S 架构构建用户端与管理端分离的业务平台，提升登记、查询与统计效率[1]。相关研究普遍以业务登记与流程梳理为核心，能够满足常规门诊与服务预约需求，但对宠物健康风险提示、服务质量评估与跨场景数据联动的深度利用仍相对有限。

在系统形态上，一些研究将领养登记、寄养预约与服务项目管理整合到统一平台，强调预约排期、订单状态与审核流转的可视化，并通过小程序或 Web 端增强用户自助操作能力[2]。同时，有方案尝试引入评价体系、公告通知与统计报表，形成“申请—审核—履约—反馈”的服务闭环，使运营人员能够基于数据进行服务调整与资源配置[3]。技术选型上，Spring Boot、MySQL 与前端组件化框架的组合逐渐成为主流，为系统的可维护性与扩展性提供支撑。

部分研究也开始关注用户评价数据与服务质量指标的关联，通过对评价标签、满意度评分与复购行为的分析，为服务改进提供量化依据[3]。但相关成果多停留在统计层面，对评价与订单履约、健康风险提示的联动机制讨论不足。

现有国内研究仍存在业务场景覆盖不均衡的问题，训练课程、健康记录与商品交易等模块往往以独立子系统存在，导致跨模块状态联动与统一数据口径难以建立；同时，权限分层、审计留痕与异常预警等治理能力相对薄弱，难以满足多角色协同与持续运营的要求[4]。在运营层面，服务过程的可视化监测、异常处置与数据驱动的精细化管理仍有提升空间。

1.3.2 国外研究现状

国外研究更强调服务流程标准化与电子健康档案建设，通过统一的数据字典与生命周期记录将预约、诊疗、寄养与随访联结为完整链路，并以 SaaS 形态向多机构提供可复用的平台能力，从而提升合规管理与跨组织协作效率[5]。相关研究通常将服务规则、档案字段与审核流程进行规范化建模，便于在多机构之间形成可共享的业务标准与质量控制体系。

围绕用户体验与运营效率，部分研究将移动端 APP、在线支付与排班优化结合，采用云端存储与 API 接口实现服务资源调度与历史记录共享，同时引入 IoT 设备或智能穿戴采集健康指标，增强服务过程的连续性与可追溯性[6]。在数据应用层面，国外系统更重视对服务偏好、复购行为与风险事件的分析，用于优化推荐策略与容量配置。

此外，国外研究在数据合规与隐私保护方面强调权限分层与审计追踪，逐步引入加密存储与访问日志，以满足多角色协作下的责任追溯需求[7]。然而，在跨系统数据共享的边界控制与实时同步方面仍存在成本与标准差异。

在商业化实践中，海外系统逐步将电商、会员订阅与评价体系嵌入服务链路，并强化隐私保护与数据授权机制，保障用户对跨平台数据流转的可控性；但在多系统互联、数据格式统一与流程一致性方面仍存在断点，跨区域协同与标准化治理仍需进一步完善[7][8]。
