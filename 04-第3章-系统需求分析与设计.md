第 3 章 系统需求分析与设计

3.1 可行性分析

宠物之家系统面向领养、寄养、服务预约、训练课程与商品交易等复合场景，业务链条涵盖信息发布、预约审核、订单履约与评价反馈，具有流程状态多、数据关联强的特点。当前宠物服务机构已具备基础网络与终端条件，系统采用前后端分离与标准化接口形式，能够在现有技术与管理条件下快速落地，并通过集中化的数据治理提升业务协同效率，因此在技术、经济与操作层面具备实施基础。

3.1.1 技术可行性

本系统后端以 Spring Boot 3.2 为核心框架，结合 MyBatis-Plus 完成数据持久化与分页查询，利用 Spring Security 与 JWT 实现身份认证与权限控制，MySQL 作为关系型数据库支撑宠物、订单、预约与评价等数据的结构化存储，技术路线成熟且社区生态完善。前端采用 Vue 3 与 Element Plus 构建管理与门户界面，配合 Axios 调用 RESTful 接口实现数据同步与状态管理，前后端职责清晰，便于扩展。系统按 Controller、Service、Mapper 分层组织业务逻辑，代码结构规范，能满足多角色访问与跨模块数据联动的技术要求。

3.1.2 经济可行性

项目选用的 Spring Boot、Vue、MyBatis-Plus 与 MySQL 均为开源技术栈，部署成本主要集中在服务器与维护人力投入，不涉及高额授权费用。系统以 B/S 架构提供服务，普通 PC 或移动终端即可访问，降低机构终端升级成本。数据库规模与访问负载适中，基础云服务器即可满足业务运行需求，后续扩容可按用户规模弹性调整，整体成本可控。

3.1.3 操作可行性

系统界面基于 Element Plus 组件库构建，表单、列表与弹窗交互符合通用管理系统习惯，用户可在较短时间内完成操作学习。普通用户可通过注册登录完成领养申请、预约寄养、服务预约与订单支付等操作，管理员通过统一后台完成数据维护与审核管理，流程直观且提示清晰。系统提供角色与菜单权限控制，能够保证不同角色的操作范围清晰可控，减少误操作风险。

3.2 系统功能需求

3.2.1 用户主要功能

用户侧覆盖游客浏览、注册登录与个人资料维护等基础功能，支持宠物信息与服务项目查询、领养申请与寄养预约提交、服务与训练课程预约、商品浏览与购物车管理、订单支付与售后申请，以及订单评价与通知消息查看。系统需要在用户操作过程中提供状态提示与记录追踪，确保预约与订单流程可回溯，满足日常服务与交易需求。

3.2.2 管理员主要功能

管理员侧主要负责平台数据的集中维护与流程审核，包括用户与角色权限管理、宠物及分类信息维护、健康记录与疫苗信息登记、领养与寄养申请审核、服务与训练课程配置、预约记录管理、商品上架与库存维护、订单与退货处理、发货信息维护、公告与轮播图发布以及统计报表查看等功能。管理员需在同一后台对多业务模块进行协调，保证服务流程与数据口径统一。

3.3 非功能性需求分析

3.3.1 性能需求

系统应支持宠物信息查询、预约提交与订单支付等高频操作的并发访问，在峰值时段保持页面响应与数据写入的稳定性。后端需通过分页查询、索引优化与接口限流保障查询效率，核心流程应在合理时间内完成处理，避免因延迟造成预约状态错乱或订单提交失败。

3.3.2 安全性需求

系统采用基于 JWT 的身份认证机制与角色权限控制策略，确保用户与管理员的操作边界清晰。密码需加密存储，关键接口应进行参数校验与权限校验，防止越权访问与非法数据提交。日志审计与异常告警机制用于记录关键操作与异常事件，配合数据库备份策略降低数据丢失风险。

3.3.3 易用性与可维护性需求

系统界面需保持导航清晰、信息层级明确，重要操作提供确认提示与结果反馈，降低用户学习成本。后端以模块化方式组织业务代码，配置与日志集中管理，便于问题定位与功能扩展。数据结构与接口命名保持一致性，便于后续维护人员理解业务逻辑并开展二次开发。
