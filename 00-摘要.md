摘  要

宠物服务机构在领养、寄养、训练、商品销售与健康记录等业务环节普遍存在信息分散、流程割裂与预约订单难以统一追踪等问题，导致用户体验不稳定、管理统计滞后与服务协同效率不足。本研究面向宠物之家业务场景，设计并实现前后端分离的信息管理系统，前端采用 Vue 3 与 Element Plus 构建门户与管理界面，后端基于 Spring Boot 3.2、MyBatis-Plus、Spring Security、JWT 与 MySQL 搭建统一服务层，覆盖用户与权限、宠物信息与健康档案、领养寄养、服务与训练预约、商品与订单、公告与统计等核心模块。系统在业务层形成从信息发布、预约申请、订单处理到评价反馈的闭环管理，并通过统一的数据字典与状态流转规则保障流程一致性。系统通过角色权限与菜单管理支撑多角色协同与操作边界控制，配合公告与内容管理提升运营支持能力。基于上述设计，系统能够提升宠物服务机构对订单履约、预约排期与健康记录的可追溯能力，为后续运营分析与服务质量管控提供稳定的数据支撑。

关键词：宠物之家；领养寄养；服务预约；Spring Boot
