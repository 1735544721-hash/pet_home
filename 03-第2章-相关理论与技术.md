第 2 章 相关理论与技术

2.1 Spring Boot 框架介绍

Spring Boot 是基于 Spring 生态的快速开发框架，提供自动配置、起步依赖与内置容器等机制，使项目能够在约定优于配置的原则下完成服务搭建，并保持配置结构的可维护性。框架集成 Spring MVC、数据访问与安全组件，支持分层架构与 RESTful 接口设计，在保障可扩展性的同时降低了环境搭建成本。通过 Starter 依赖统一版本管理，系统在引入邮件、文件上传等组件时无需复杂配置，减少了依赖冲突与维护负担。宠物之家系统后端以 Spring Boot 3.2 为核心，结合 MyBatis-Plus、Spring Security 与 JWT 实现统一的服务入口、权限控制与数据持久化，并通过模块化的 Controller 与 Service 组织宠物、订单、服务预约等业务逻辑，保证接口一致性与扩展性。借助配置绑定与多环境 profile，系统能够灵活切换开发与生产参数，同时结合 AOP 与日志机制完成异常处理与审计记录，为前后端分离部署提供稳定的基础服务。

2.2 Vue.js 前端框架

Vue.js 是渐进式前端框架，采用组件化与响应式数据驱动机制，配合虚拟 DOM 提升界面渲染效率，适合构建交互密集型管理系统。

本系统前端以 Vue 3 为核心，结合 Vue Router 实现路由组织与权限页面切换，利用 Pinia 管理用户登录态与菜单数据，确保不同角色获得对应的功能入口。界面层使用 Element Plus 组件库，快速搭建宠物信息、服务预约与订单管理等页面，并通过表单校验、分页与弹窗交互提升可用性。与此同时，Axios 负责与后端 REST 接口的数据交互，形成统一的业务请求与错误处理流程，保证前端展示与后台状态同步。

2.3 MyBatis-Plus 数据访问框架

MyBatis-Plus 是在 MyBatis 基础上进行功能增强的持久层框架，通过通用 CRUD、条件构造器与插件体系降低手写 SQL 与配置成本，同时保持原有 MyBatis 的灵活性与可控性。其核心优势在于对实体类与数据表的自动映射、对常见增删改查的封装以及对分页、逻辑删除与性能优化插件的支持，使开发者能够用更少的样板代码完成稳定的数据访问层搭建。

宠物之家系统在数据访问层采用 MyBatis-Plus 组织 Mapper 与 Service 结构，实体类映射用户、宠物、领养寄养、服务预约、商品订单等业务表，Mapper 继承 BaseMapper 提供统一接口，Service 层通过 LambdaQueryWrapper 与分页 Page 对象完成筛选、排序与分段查询。系统在列表检索与状态筛选场景中复用条件构造器，配合自定义 SQL 与多表查询实现统计与关联数据加载，在保证查询效率的同时保持代码结构清晰，便于后续扩展与维护。

2.4 MySQL 数据库

MySQL 是成熟的关系型数据库管理系统，基于表结构与主外键约束构建数据一致性模型，具备事务机制、索引优化与稳定的并发处理能力，适合承载业务数据长期存储与多维度检索需求。在宠物之家系统中，MySQL 作为核心数据仓库，承担用户权限、宠物信息、领养寄养记录、服务预约、商品与订单、评价反馈及公告统计等数据的结构化存储，通过外键关联与索引设计支撑跨模块查询与统计分析。系统针对订单编号、用户账号与宠物编号等高频检索字段建立索引，结合事务保证预约与订单状态变更的原子性，从而提高数据一致性与业务处理可靠性。

2.5 Java 语言

Java 是面向对象的通用编程语言，具有跨平台、类型安全与完善的类库生态，能够支持大型应用的工程化开发与长期维护。其虚拟机机制使应用在不同操作系统之间具备良好的可移植性，集合框架、并发工具与网络通信类库为高并发与多角色系统提供稳定的基础能力。宠物之家系统后端以 Java 为开发语言，结合 Spring Boot 3.2 构建分层架构，利用注解驱动与依赖注入组织控制层、业务层与数据访问层，形成清晰的模块边界。Java 的异常处理机制与日志体系用于统一的错误捕获与审计记录，配合 Bean 校验、枚举与工具类增强数据输入的可靠性与可读性，使系统在用户注册、预约下单、状态流转等关键流程中保持稳定与可追溯的业务行为。
